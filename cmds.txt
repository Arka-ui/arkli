- arkli init [the name of the website] : 
    - creates a new project
    - create isolated directory for the databse and the .env files

- arkli move [the name of the website] : 
    - moves the files/folders into the direcotry that is created by arkli init
    - auto detect .env file and move it to the new directory
    - auto detect database and move it to the new directory
    (need to be run in the temp folder where only the actual website is stored then it delete it to leave no trace)

- arkli migrate -d [path to a specific database] : 
    - auto migrate the database to the latest version

- arkli mail -setup [domain name of the website] : 
    - setup the mail configuration with open port and all the needed files + all the needed configurations

- arkli mail -create [name for the address] -d [domain name of the website] : 
    - create a new mail address for a specific website
    - give the user the needed information to set it up and use it (passwords + ports + addresses)

- arkli mail -delete [name for the address] -d [domain name of the website] : 
    - delete a mail address for a specific website

- arkli mail -list [domain name of the website] : 
    - list all the mail addresses for a specific website

- arkli mail -info [name for the address] -d [domain name of the website] : 
    - give the user the needed information to set it up and use it (passwords + ports + addresses)

- arkli cert -setup [domain name of the website] : 
    - setup the certificate for a specific website
    - auto detect the domain name and setup the certificate
    - (we add a -a to the command if we want the cert to be auto renewed)
    
- arkli cert -delete [domain name of the website] : 
    - delete the certificate for a specific website
    - auto detect the domain name and delete the certificate

- arkli cert -list [domain name of the website] : 
    - list all the certificates for a specific website
    - auto detect the domain name and list the certificates

(notes : for the mail system every config need that to acces the mail server we use a mail.[domain name])
